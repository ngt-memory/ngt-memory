## Что такое NGT Memory?

NGT Memory даёт LLM **персистентную кросс-сессионную память** — аналог гиппокампа для ИИ.

Вместо того чтобы забывать всё между разговорами, LLM запоминает:
- Предпочтения пользователя, аллергии, ограничения
- Прошлые решения и контекст
- Кросс-доменные факты (медицина + питание + путешествия — всё связано)

**Результаты бенчмарков (Exp 44, gpt-4o-mini):**

| Режим | Фактуальная согласованность (0-3) | Совпадение ключевых слов |
|-------|----------------------------------|--------------------------|
| **NGT Memory (граф)** | **2.33 / 3** | **44%** |
| **NGT Memory (эмбеддинги)** | **2.44 / 3** | **44%** |
| Без памяти | 1.22 / 3 | 27% |
| **Δ улучшение** | **+1.22 (+100%)** | **+17 п.п.** |

Пример — **без памяти**:
> Пользователь: «Какие рестораны в Киото посоветуете?»
> LLM: *«Ippudo — отличный рамен»* ← порекомендовал мясной рамен вегетарианцу ❌

**С NGT Memory** (помнит, что пользователь вегетарианец):
> LLM: *«Shigetsu при храме Тэнрю-дзи — сёдзин рёри (буддийская вегетарианская кухня)»* ✓

## Самостоятельная установка

### Вариант 1: Локально / для разработки

```bash
# 1. Клонировать
git clone https://github.com/ngt-memory/ngt-memory
cd ngt-memory

# 2. Установить зависимости
pip install -r requirements.txt -r requirements-api.txt

# 3. Настроить
cp .env.example .env
# Открыть .env, указать: OPENAI_API_KEY=sk-...

# 4. Запустить
uvicorn api.main:app --host 0.0.0.0 --port 9190 --reload
```

API: http://localhost:9190
Swagger UI: http://localhost:9190/docs

### Вариант 2: Docker (рекомендуется)

```bash
cp .env.example .env
# Указать OPENAI_API_KEY в .env

docker-compose up -d
```

> **Примечание:** по умолчанию 1 воркер `uvicorn` — это сделано намеренно.
> Сессии хранятся в оперативной памяти, один воркер обеспечивает консистентность `session_id`.
> Для многопоточности используйте sticky-маршрутизацию или общий backend сессий.

### Вариант 3: VPS (Ubuntu/Debian)

```bash
git clone https://github.com/ngt-memory/ngt-memory
cd ngt-memory
cp .env.example .env
nano .env   # указать OPENAI_API_KEY

docker-compose up -d
```

API будет доступен на порту **9190**. Для HTTPS поставьте nginx перед ним.

## Справочник API

### `POST /chat` — Диалог с памятью

Отправляет сообщение в LLM. Автоматически извлекает релевантные воспоминания и инжектирует их в контекст. Сохраняет разговор в память.

**Запрос:**
```json
{
  "message": "Какие лекарства мне нельзя принимать?",
  "session_id": "user_42",
  "use_memory": true
}
```

**Ответ:**
```json
{
  "response": "Учитывая вашу аллергию на пенициллин и приём лизиноприла, вам следует избегать НПВС...",
  "session_id": "user_42",
  "memories_used": [
    {"text": "У пациента аллергия на пенициллин", "score": 0.91},
    {"text": "Принимает лизиноприл 10 мг ежедневно", "score": 0.87}
  ],
  "memories_count": 2,
  "tokens_in": 312,
  "tokens_out": 89,
  "latency_ms": 1847.3,
  "memory_entries": 14
}
```

**Пример cURL:**
```bash
curl -X POST http://localhost:9190/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Меня зовут Антон, у меня аллергия на пенициллин.", "session_id": "demo"}'
```

### `POST /store` — Сохранить факт напрямую

Загрузка фактов в память без диалога (профили пользователей, документы, структурированные данные).

**Запрос:**
```json
{
  "text": "Пациент Антон, 1990 г.р., аллергия на пенициллин, принимает лизиноприл 10 мг.",
  "session_id": "medical_user_1",
  "domain": "medical"
}
```

**Ответ:**
```json
{
  "success": true,
  "session_id": "medical_user_1",
  "memory_entries": 1,
  "message": "Stored. Total memory entries: 1"
}
```

---

### `POST /retrieve` — Семантический поиск по памяти

Прямой семантический поиск без вызова LLM. Для построения собственных пайплайнов с дополненной памятью.

**Запрос:**
```json
{
  "query": "лекарственные аллергии",
  "session_id": "medical_user_1",
  "top_k": 5,
  "use_graph": true,
  "threshold": 0.2
}
```

**Ответ:**
```json
{
  "results": [
    {
      "text": "У пациента аллергия на пенициллин",
      "score": 0.91,
      "domain": "medical",
      "concepts": ["penicillin", "allergy", "patient"]
    }
  ],
  "count": 1,
  "session_id": "medical_user_1",
  "query": "лекарственные аллергии"
}
```

## Модель сессий

Каждый `session_id` имеет **изолированную память**. Используйте разные session_id для разных пользователей:

```
session_id="user_42"   → память пользователя 42
session_id="user_99"   → отдельная, изолированная память
session_id="default"   → общая (для однопользовательских систем)
```

Сессии истекают после `NGT_SESSION_TTL` секунд неактивности (по умолчанию: 1 час).

Важно при развёртывании:

- С текущим in-memory `SessionStore` Docker запускает **1 воркер uvicorn**.
- Для многопоточного деплоя нужны **sticky-сессии** или **общий backend сессий**.
- Без этого один `session_id` может быть разделён между процессами.

## Пример клиента на Python

```python
import httpx

BASE_URL = "http://localhost:9190"
SESSION = "my_user"

client = httpx.Client(base_url=BASE_URL)

# Первый разговор — представляемся
r = client.post("/chat", json={
    "message": "Я вегетарианец и планирую поездку в Киото в следующем месяце.",
    "session_id": SESSION
})
print(r.json()["response"])

# На следующий день — новый вопрос, память сохранилась
r = client.post("/chat", json={
    "message": "Какие рестораны посоветуете в Киото?",
    "session_id": SESSION
})
# → Рекомендует вегетарианские рестораны, помнит поездку в Киото
print(r.json()["response"])
print(f"Использовано воспоминаний: {r.json()['memories_count']}")
```

## Конфигурация

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `OPENAI_API_KEY` | *обязательно* | Ключ OpenAI API |
| `NGT_API_SECRET` | *(пусто)* | Опциональный ключ защиты API |
| `NGT_CHAT_MODEL` | `gpt-4.1-nano` | Chat-модель OpenAI (также gpt-5-nano, o4-mini и др.) |
| `NGT_EMBEDDING_MODEL` | `text-embedding-3-small` | Модель эмбеддингов |
| `NGT_MEMORY_TOP_K` | `5` | Количество воспоминаний на каждый запрос |
| `NGT_MEMORY_THRESHOLD` | `0.25` | Минимальный cosine score для включения воспоминания |
| `NGT_USE_GRAPH` | `true` | Включить граф-усиленное извлечение |
| `NGT_SESSION_TTL` | `3600` | Таймаут неактивности сессии (секунды) |
| `NGT_MAX_SESSIONS` | `100` | Максимум одновременных сессий |
| `NGT_CORS_ORIGINS` | `*` | Допустимые CORS-источники через запятую |
| `NGT_LOG_LEVEL` | `INFO` | Уровень логирования |
| `NGT_LOG_JSON` | `true` | Структурированные JSON-логи |

## Архитектура

```
Запрос пользователя
     ↓
[POST /chat]
     ↓
OpenAI Embeddings (text-embedding-3-small)  ~700 мс
     ↓
NGT Memory Retrieve (cosine + граф)         ~2-3 мс
     ↓
Системный промпт + инъекция [MEMORY CONTEXT]
     ↓
OpenAI Chat (gpt-4.1-nano)                 ~800-1500 мс
     ↓
Сохранение в NGT Memory                    ~1 мс
     ↓
Ответ
```

**Накладные расходы NGT: ~2-3 мс** (без учёта вызовов API).
Все вызовы OpenAI полностью асинхронные (`AsyncOpenAI`) — параллельная обработка запросов.
Docker по умолчанию: **1 воркер uvicorn** для консистентности in-memory сессий.

Поддерживаемые модели: `gpt-4.1-nano` (быстрейшая), `gpt-4.1-mini`, `gpt-5-nano` (reasoning), `o4-mini`.

## Как работает NGT Memory

NGT Memory объединяет **три механизма извлечения**:

1. **Косинусное сходство** — находит семантически близкие факты
2. **Хеббовский граф ассоциаций** — связывает концепции, появляющиеся вместе
   (например, «вегетарианец» ↔ «ресторан» ↔ «Киото» связываются после первого упоминания)
3. **Иерархическая консолидация** — важные факты повышаются до семантической памяти

Это обеспечивает **кросс-сессионное вспоминание**: вопрос о «ресторанах» → извлекает «вегетарианские предпочтения», даже если слова не пересекаются.

| Операция | Пропускная способность | Задержка (p50) |
